# Indigold — v0.1 (React PWA)

A mobile-first **Personal Intelligence Operating System** prototype, built with
React 19 + Vite 7 + TypeScript + Tailwind 4, themed as a **Deep Space Observatory**.

> **Cardinal Rule:** This prototype runs **exclusively on synthetic data**
> (`client/public/data/*.json`). No real vault, no API keys, no real user data.
> Maintain this separation until an explicit "connect real vault" directive.

## Stack
React 19 · TypeScript 5.6 · Tailwind CSS 4 (OKLCH) · Wouter · Lucide · Sonner ·
Vite 7 · pnpm. Bottom-tab PWA with seven views.

## Commands
```sh
pnpm install
pnpm dev       # Vite dev server on http://localhost:3000
pnpm check     # tsc --noEmit (typecheck)
pnpm test      # jsdom runtime smoke test — mounts <App/> across all 7 routes
pnpm build     # production build -> dist/public
pnpm start     # serve dist/public via Express (http://localhost:3000)
```

## Views (bottom tab bar)
Home (Dashboard) · Inbox · Timeline · Atlas (Liminal Atlas graph) · Context
(Encompass) · Brief (Radian) · I/O (Import/Export). See the build handoff for
per-view detail.

## Offline policy — HYBRID
Per the build decision, this app uses a **hybrid** offline strategy:
- **Service worker** (`client/public/sw.js`) precaches the app shell + all
  synthetic fixtures and images, and stale-while-revalidates Google Fonts so the
  app works offline after first load.
- **Fonts** load from Google Fonts CDN on first run (network). *Path to fully
  local:* self-host the Space Grotesk / JetBrains Mono `.woff2` files under
  `client/public/fonts/` and swap the `<link>` in `client/index.html` for
  `@font-face` rules.
- **Images** are **local** (`client/public/images/*.png`), generated by
  `scripts/generate_assets.py` (pure Python, no deps). The handoff's CDN image
  URLs were wildcard placeholders and are intentionally not used.

## Regenerating assets
```sh
python3 scripts/generate_assets.py   # icons + cosmic backgrounds
```

## Capture workflow (iPhone-first)
Indigold is the **mobile mission-control** surface; **Apple Shortcuts** are the
capture bridge. Capture fast on iPhone → review in the PWA → process later with AI
→ promote only valuable items into memory. The Inbox renders nine synthetic
capture types (`apple_note`, `web_link`, `instagram_reel`, `threads_post`,
`screenshot`, `voice_memo`, `document`, `llm_conversation`, `manual_text`) with
type/source/sensitivity badges, processing status, and quick triage. v0.1 has **no
capture automation, no OneDrive/iCloud API, and no AI processing** — the real vault
stays separate. See [`docs/CAPTURE_WORKFLOW.md`](docs/CAPTURE_WORKFLOW.md) and the
Shortcut output examples in [`docs/sample_captures/`](docs/sample_captures).

## Data schema
`client/src/lib/types.ts` defines the contracts; fixtures live in
`client/public/data/`. Nodes carry a single `truth_layer` (A–F) and an `mvs`
(Memory Value Score 0–100). Edges carry `valid_from` temporal validity.

## Guardrails
No external API/data calls beyond Google Fonts (documented above). No telemetry,
no analytics, no cloud sync, no secrets. The `server/` directory is a static-file
compatibility placeholder and connects to nothing.
